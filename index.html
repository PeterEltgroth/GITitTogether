What is version control?

A system that:

  • records changes over time
  • allows you to revert versions
  • does comparisons across time
  • knows who changed what and when it was changed
    • About Version Control

    Local Version Control Systems

    What is good and bad about this design?

    Could do this manually by copying and versioning files/directories, but that's error prone.
    Pros: At least you have versioning.
    Cons: Can't easily share across computers.
    Example: VMS automates this eliminating some human error by appending a version number to the file name.

    Centralized Version Control Systems

    What are the pros and cons of this?

    A single server holds all the versioned files and clients connect to it and check them out. Pros: Can share across computers and administrators can control priviledges.
    Cons: There is single point of failure. Clients have to be connected to the server to see history and share.
    Examples: CVS, Subversion, Perforce, ClearCase

    Distributed Version Control Systems

    What do you like and not like about this?

    Pros: Everyone has a full copy of the repository, built in backup!
    Cons: More complicated to learn.
    Examples: GIT, Mercurial, Bazaar, Darcs
